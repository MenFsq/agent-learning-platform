# Agent Learning Platform Progress

**更新日期**: 2026-04-16

## 进度原则

这份进度文档不再使用脱离代码现状的整体百分比。

原因很简单：当前项目最核心的目标是 `LangChain Agent` 集成，而这一部分还没有真正落地。如果 AI 主链路还没写出来，写“整体 65%”这类数字会显得不真实，也不利于判断优先级。

因此，这里改为使用“模块状态 + 阻塞项 + 下一步”的方式记录进展。

## 当前阶段判断

项目目前处于：

**前后端基础已搭建，Agent 集成尚未开始落地的阶段。**

更准确地说：

- UI 和学习内容已经有明显进展
- 后端基础框架已经成型
- 但真正决定项目价值上限的 `LangChain Agent` 集成，当前仍基本为 `未实现`

## 模块状态

### 1. 前端工作台

**状态：已初步完成**

已看到的内容包括：

- 仪表板页面和多个工作台组件
- 学习页与 `LangChain` 知识地图
- 全局导航、主题切换、通知等交互外壳
- 基础路由和页面组织

说明：前端已经不是空壳，但目前更偏展示层和学习工作台，不是 Agent 产品闭环。

### 2. 后端应用骨架

**状态：已搭建**

已看到的内容包括：

- `FastAPI` 主应用入口
- 中间件、异常处理、健康检查
- 认证、项目、学习等 API 模块
- 数据库配置、模型目录、服务目录

说明：后端框架具备继续开发的条件，但不少接口仍是骨架或模拟实现。

### 3. 认证与业务接口

**状态：部分完成**

现状判断：

- 有路由和数据模型
- 有 `JWT` 相关能力
- 但部分用户查询、用户创建、资料获取仍然是模拟逻辑
- 项目管理接口存在较多内存数据和演示性质内容

说明：可以作为联调基础，但还不能视为成熟业务能力。

### 4. LangChain / Agent 集成

**状态：未真正开始**

当前只确认到：

- `backend/requirements.txt` 中声明了 `langchain`
- 仓库中有 `LangChain` 学习内容、文案和若干说明
- 文档多次提到 AI 集成方向

当前没有确认到的核心实现包括：

- Agent 执行器
- tool calling
- prompt 编排服务
- memory / session 上下文管理
- AI chat API 的真实后端实现
- trace / callback / evaluation 体系

这部分是项目当前最大的缺口，也是最关键的主线任务。

## 已完成事项

### 已完成

- 前端工程已初始化并形成较完整工作台界面
- 学习模块已有较成熟的 `LangChain` 知识地图内容
- 后端工程已具备 `FastAPI` 基础结构
- 基础认证、安全、中间件、数据库配置已接入到工程骨架
- 前后端本地联调基础条件已经具备

### 尚未完成

- 真实可运行的 Agent 对话链路
- 面向 Agent 的独立后端模块拆分
- AI 任务执行、工具编排、结果持久化
- 前端到后端的真实 AI 交互闭环
- 与 Agent 相关的测试和观测能力

## 当前最重要的问题

### 1. 项目主价值尚未落地

仓库主题是 Agent Learning Platform，但真正决定差异化的 `LangChain Agent` 集成还没有实现。

### 2. 文档表述曾明显超前

之前一些文档把“依赖已声明”写成“能力已集成”，把“预留接口”写成“功能已完成”。这会让后来者错误判断项目成熟度。

### 3. 当前工作重心容易被 UI 稀释

前端已经比较显眼，但项目最难、也最关键的部分其实是后端 AI 能力层。如果不尽快补这块，项目会长期停留在“展示性平台”。

## 下一阶段计划

### 第一优先级

先打通最小可运行的 `LangChain Agent` 闭环：

1. 新增后端 `agent` 模块
2. 设计一个最小 `AgentService`
3. 实现一个 `/api/v1/agent/chat` 接口
4. 接入一个模型和 1 到 2 个真实 tools
5. 保存消息、调用记录和错误日志

### 第二优先级

完成前端与真实 Agent 接口联调：

1. 增加 Agent 对话或任务入口
2. 展示执行状态、响应内容和失败信息
3. 支持基础多轮上下文

### 第三优先级

补齐工程化能力：

1. trace / callback / logging
2. Prompt 配置管理
3. 基础测试
4. 错误恢复与限流

## 当前结论

如果必须给一句真实判断：

> 这个项目已经完成了“平台外壳”和“学习内容工作台”的大半基础，但还没有进入“LangChain Agent 已集成”的阶段。

所以接下来最重要的不是继续美化进度数字，而是尽快把第一个真实 Agent 用例跑起来。
